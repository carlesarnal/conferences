Real-time data processing is critical in domains like finance, social media, and cybersecurity. In this talk, I’ll present a fully open source, end-to-end system for real-time content classification, deployed on Kubernetes. Using Reddit flair classification as a case study, the pipeline combines Apache Kafka for ingestion, Apache Spark Streaming for low-latency processing, and a machine learning model for classification, with predictions exposed through a Quarkus-based API.

This session goes beyond architecture to show the system in action: how Reddit posts are streamed into Kafka, classified in Spark, and served via microservices. I’ll also cover practical deployment aspects on Kubernetes—scaling, monitoring, and extending the pipeline to other event-classification tasks. Attendees will gain hands-on insights into building and operating streaming ML systems on Kubernetes using only open source technologies.
