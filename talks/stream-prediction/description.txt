Title: Real-Time Content Classification on Kubernetes: An End-to-End Open Source ML Pipeline

A Reddit post is published. Before a human moderator even sees it, our pipeline has ingested it through Kafka, classified it with a dual-model ML inference in Spark, and served the prediction through a Quarkus API — all running on Kubernetes, all fully open source.

In this session, I’ll demo this end-to-end system live and walk through every layer: Reddit API ingestion into Kafka, Spark Structured Streaming for low-latency dual-model classification, a Quarkus microservice exposing predictions, and Prometheus/Grafana dashboards monitoring the entire pipeline. No slides-only architecture — you’ll see real posts classified in real time.

Attendees will leave with:
- A complete, deployable blueprint for streaming ML inference on Kubernetes
- Practical guidance on scaling Spark Streaming jobs and handling backpressure in production
- Patterns for extending the pipeline to other event-classification domains (fraud detection, content moderation, log analysis)
